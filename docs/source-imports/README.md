# Source Imports

Curated provenance summaries for zip bundles that influenced the repo.

These files are not shipped product docs. They exist so imported bundle content keeps a traceable origin after the useful parts are folded into `docs/`, `examples/`, and `.beads`.

Raw extracted bundle trees are intentionally not checked in here. They create formatting, lint, and spell noise without adding durable product value.

## How to use this area

- Treat these files as provenance notes, not as the canonical product story.
- Prefer the curated docs under `docs/evaluation/`, `docs/strategy/`, `docs/wip/`, and the tracked backlog in `.beads/issues.jsonl`.
- When a raw pack matters long-term, add or update a short summary here and put the accepted content in the right user-facing docs folder.
