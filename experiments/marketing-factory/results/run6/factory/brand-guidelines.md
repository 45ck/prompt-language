# CloudPulse Brand Guidelines

## Brand Overview

**CloudPulse** is a cloud monitoring SaaS platform built for engineering teams and DevOps professionals. Our brand communicates reliability, speed, intelligence, and modern technical sophistication.

**Tagline:** _Monitor smarter, respond faster._

**Mission:** Empower engineering teams with real-time cloud monitoring that turns data into action.

---

## Voice and Tone

### Voice Attributes

- **Confident but not arrogant** — We know our product is powerful; we let the value speak for itself.
- **Technical but accessible** — We speak the language of engineers without alienating newcomers.
- **Direct and concise** — DevOps professionals value clarity. No fluff, no jargon for jargon's sake.
- **Empowering** — We position the user as the hero; CloudPulse is the tool that makes them unstoppable.

### Tone by Context

| Context              | Tone                                 |
| -------------------- | ------------------------------------ |
| Marketing / Hero     | Bold, aspirational, energetic        |
| Feature descriptions | Clear, precise, benefit-focused      |
| Pricing              | Transparent, straightforward, fair   |
| Testimonials         | Authentic, relatable, results-driven |
| FAQ / Support        | Helpful, patient, reassuring         |
| Error states         | Calm, solution-oriented              |

### Writing Principles

1. Lead with the benefit, not the feature.
2. Use active voice.
3. Keep sentences short — aim for 15 words or fewer.
4. Use "you" and "your" to speak directly to the reader.
5. Avoid superlatives unless backed by data.

## Color Palette

### Primary Colors

| Role      | Color  | Hex       | Usage                                                      |
| --------- | ------ | --------- | ---------------------------------------------------------- |
| Primary   | Blue   | `#2563EB` | CTAs, links, primary buttons, key UI elements              |
| Secondary | Purple | `#7C3AED` | Gradients, accents, secondary actions, decorative elements |
| Accent    | Green  | `#10B981` | Success states, highlights, badges, positive indicators    |

### Neutral Colors

| Role        | Hex       | Usage                         |
| ----------- | --------- | ----------------------------- |
| White       | `#FFFFFF` | Backgrounds, text on dark     |
| Light Gray  | `#F8FAFC` | Section backgrounds           |
| Medium Gray | `#64748B` | Body text, secondary text     |
| Dark Gray   | `#1E293B` | Headings, primary text        |
| Near Black  | `#0F172A` | Dark mode backgrounds, footer |

### Gradient

- **Primary gradient:** `linear-gradient(135deg, #2563EB, #7C3AED)` — Used for hero backgrounds, CTAs, and decorative elements.

### Dark Mode

- Background: `#0F172A`
- Surface: `#1E293B`
- Text: `#F1F5F9`
- All brand colors remain the same; adjust opacity for subtlety.

## Typography

### Font Stack

```
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif;
```

### Type Scale

| Element         | Size            | Weight | Line Height |
| --------------- | --------------- | ------ | ----------- |
| H1 (Hero)       | 3.5rem / 56px   | 800    | 1.1         |
| H2 (Section)    | 2.25rem / 36px  | 700    | 1.2         |
| H3 (Card title) | 1.25rem / 20px  | 600    | 1.3         |
| Body            | 1.125rem / 18px | 400    | 1.6         |
| Small / Caption | 0.875rem / 14px | 400    | 1.5         |
| Button          | 1rem / 16px     | 600    | 1           |

### Mobile Adjustments

- H1 reduces to 2.5rem
- H2 reduces to 1.75rem
- Body remains 1.125rem

## Naming Conventions

### Product Names

- **CloudPulse** — Always one word, capital C and P. Never "Cloud Pulse" or "cloudpulse."
- **CloudPulse Pro / CloudPulse Enterprise** — Tier names follow the product name.

### Feature Names

- Use descriptive, plain-language names: "Real-Time Alerts," "Custom Dashboards," "Intelligent Routing."
- Do not trademark individual features.

### CTA Labels

- Primary: "Start Free Trial" or "Get Started Free"
- Secondary: "Book a Demo" or "Learn More"
- Avoid generic "Submit" or "Click Here."

---

## Imagery and Iconography

- Use clean, modern line icons with 2px stroke.
- Illustrations should feel technical but approachable — think dashboards, graphs, cloud infrastructure.
- Avoid stock photos of people pointing at screens.
- Icon color: Primary blue `#2563EB` or white on dark backgrounds.

---

## Spacing and Layout

- Use an 8px grid system.
- Section padding: 80px vertical (desktop), 48px (mobile).
- Max content width: 1200px, centered.
- Card border-radius: 12px.
- Button border-radius: 8px.
